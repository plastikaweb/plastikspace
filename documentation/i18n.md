Quick setup guide for adding translations to any Angular app using ngx-translate.

## Installation

```bash
yarn add @ngx-translate/core @ngx-translate/http-loader
```

## Directory Structure

Create translation files in your app:

```bash
src/
├── assets/
│   └── i18n/
│       ├── en.json
│       ├── ca.json
│       └── es.json
├── app/
└── ...
```

## Configure in bootstrapApplication

Add to your `main.ts`:

```typescript
import { bootstrapApplication } from '@angular/platform-browser';
import { HttpClient, provideHttpClient } from '@angular/common/http';
import { provideTranslateService, TranslateLoader } from '@ngx-translate/core';
import { TranslateHttpLoader } from '@ngx-translate/http-loader';

import { AppComponent } from './app/app.component';

export function HttpLoaderFactory(http: HttpClient) {
  return new TranslateHttpLoader(http, './assets/i18n/', '.json');
}

bootstrapApplication(AppComponent, {
  providers: [
    provideHttpClient(),
    provideTranslateService({
      loader: {
        provide: TranslateLoader,
        useFactory: HttpLoaderFactory,
        deps: [HttpClient],
      },
      defaultLanguage: 'en',
    }),
  ],
}).catch(err => console.error(err));
```

## Initialize Language in App Component

```typescript
import { Component, OnInit, inject } from '@angular/core';
import { TranslateService } from '@ngx-translate/core';

@Component({
  selector: 'app-root',
  template: `<h1>{{ 'app.title' | translate }}</h1>`,
})
export class AppComponent implements OnInit {
  private translateService = inject(TranslateService);

  ngOnInit(): void {
    this.translateService.setDefaultLanguage('en');
    const browserLang = this.translateService.getBrowserLanguage();
    const lang = ['ca', 'es'].includes(browserLang) ? browserLang : 'en';
    this.translateService.use(lang);
  }
}
```

## Use in Templates

```html
<!-- Simple key -->
<h1>{{ 'app.title' | translate }}</h1>

<!-- With parameters -->
<p>{{ 'messages.greeting' | translate: { name: 'John' } }}</p>

<!-- In attributes -->
<button [title]="'common.save' | translate">Save</button>
```

## Use in Components

```typescript
import { Component, OnInit, inject } from '@angular/core';
import { TranslateService } from '@ngx-translate/core';

@Component({
  selector: 'app-example',
  template: `<p>{{ message }}</p>`,
})
export class ExampleComponent implements OnInit {
  private translateService = inject(TranslateService);
  message: string = '';

  ngOnInit(): void {
    // Synchronous (immediate)
    this.message = this.translateService.instant('common.success');

    // Asynchronous (recommended)
    this.translateService.get('common.error').subscribe(msg => {
      console.log(msg);
    });
  }

  changeLanguage(lang: string): void {
    this.translateService.use(lang);
  }
}
```

## Use in Services

```typescript
import { Injectable, inject } from '@angular/core';
import { TranslateService } from '@ngx-translate/core';

@Injectable({
  providedIn: 'root',
})
export class NotificationService {
  private translateService = inject(TranslateService);

  showMessage(key: string): void {
    const message = this.translateService.instant(key);
    console.log(message);
  }
}
```

## Validation

The project includes automatic validation of translation keys:

```bash
# Validate all translation files
yarn i18n:validate

# Output:
# ✓ Loaded 45 keys from en.json
# ✓ Found 120 files to check
# ✅ All translation keys are valid!
```

The validation script:

- Scans all `.html` and `.ts` files
- Extracts translation keys from templates and code
- Verifies all keys exist in the source language file
- Runs automatically on pre-commit and CI

## Key Naming Convention

Use dot notation and keep it organized:

```json
{
  "feature": {
    "component": {
      "title": "...",
      "description": "..."
    }
  }
}
```

Access: `{{ 'feature.component.title' | translate }}`

## Troubleshooting

### Translation shows key name instead of text

- Check path in `TranslateHttpLoader` matches your directory
- Verify language code in `use()` matches filename (e.g., 'ca' for ca.json)
- Check browser console for load errors

### Missing keys error

- Ensure key exists in en.json
- Run `yarn i18n:validate` to find which keys are missing
- Add key to ALL language files

### Build fails

- Ensure i18n files are in `src/assets/i18n/`
- Run `yarn i18n:validate` to verify setup

## Test Suite

Run tests to verify the validation system:

```bash
yarn i18n:test
```

**When to run:**

- After modifying translation files
- Before committing changes
- In CI/CD pipeline
- After modifying validation script logic

**What it tests:**

- Valid keys in templates
- TypeScript pattern detection
- Missing keys detection
- Multiple apps with different languages
- Alternative directory structures
- Keys with parameters
- Source language selection
